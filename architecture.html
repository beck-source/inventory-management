Filter Data Flow — End to End

1
User selects filter in FilterBar.vue
Dropdown changes update reactive refs in the useFilters.js composable singleton — shared state across all views.
selectedPeriod.value = '2025-01'  ·  selectedLocation.value = 'San Francisco'
2
watch() in each view triggers a data reload
Every view watches the filter refs. When any filter changes, it calls getCurrentFilters() and re-fetches from the API.
watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus], loadData)
3
api.js builds query params and calls FastAPI
Non-'all' filter values are appended as URL query parameters. 'all' values are skipped entirely.
GET /api/orders?warehouse=San+Francisco&category=Sensors&status=Delivered&month=2025-01
4
FastAPI applies filters in Python
apply_filters() runs list comprehensions for warehouse, category, status. filter_by_month() matches month substring or maps quarter to 3-month range.
filtered = [o for o in orders if o['warehouse'] == warehouse]
5
Pydantic validates and serializes the response
FastAPI validates filtered results against the List[Order] response model before sending JSON.
response_model=List[Order] → JSON array
6
Vue computed properties derive display data from refs
Raw data stored in refs (allOrders, inventoryItems). Computed properties derive charts, summaries, and filtered tables automatically.
const totalValue = computed(() => orders.value.reduce((s, o) => s + o.total_value, 0))

Filter System

Time Period
?month=
2025-01 through 2025-12 · Q1–Q4 quarters · "all"
Warehouse / Location
?warehouse=
San Francisco · London · Tokyo · "all"
Category
?category=
Circuit Boards · Sensors · Actuators · Controllers · Power Supplies · "all"
Order Status
?status=
Delivered · Shipped · Processing · Backordered · "all"
Filters applied: inventory supports warehouse + category only (no time dimension). spending endpoints have no filters. demand/backlog have no filters.
