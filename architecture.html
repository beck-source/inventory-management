Request Data Flow
1
User changes a filter
FilterBar.vue updates a ref in the singleton useFilters composable. The same ref instance is shared across all views.
useFilters() → selectedLocation.value = 'Tokyo'
2
View reacts and fetches
Each view watches the filter composable. A watcher calls the relevant api.js function, passing current filter values as query params.
api.getInventory({ warehouse: 'Tokyo', category: 'all' })
3
FastAPI receives request
The endpoint reads query params. If a param is 'all' or missing, that filter is skipped. Filtering runs on the in-memory list loaded at startup.
GET /api/inventory?warehouse=Tokyo
4
Pydantic validates & serializes
Results are passed through a Pydantic response model. Any field mismatch between the model and JSON data raises a validation error at startup.
response_model=List[InventoryItem]
5
Vue stores raw data in refs
The API response is written to a raw ref (e.g. inventoryItems). Computed properties that depend on it automatically recalculate.
inventoryItems.value = response.data
6
Template re-renders
Computed properties (filtered lists, aggregates, chart data) derived from the refs update, Vue diffs the DOM, and the UI updates in place. No full reload.
computed(() => inventoryItems.value.filter(...))

Filter System — Query Parameters
Endpoint Group warehouse category status month Notes
/api/inventory ✓ ✓ — — No time dimension
/api/orders ✓ ✓ ✓ ✓ Most flexible endpoint
/api/dashboard/summary ✓ ✓ ✓ ✓ Aggregates inventory + orders
/api/demand, /api/backlog — — — — Always returns all records
/api/reports/quarterly — — — — Format: Q1-2025, Q2-2025…

Passing 'all' for any parameter skips that filter. Revenue targets: $800K/month (single month), $9.6M YTD (all months).
