Factory Inventory Management System

A full-stack inventory and operations demo: a Vue 3 single-page application backed by a Python FastAPI service that serves in-memory mock data loaded from JSON files. No database — all state lives in process memory for the lifetime of the server.

Data Flow — A Filtered Request

1

User adjusts a filter

One of four filters changes in the FilterBar: Time Period, Warehouse, Category, or Order Status.

2

Shared filter state updates

The useFilters composable holds filter state as module-level refs (a singleton shared across all views). getCurrentFilters() maps Period → month and Location → warehouse.

3

API client builds the request

api.js serializes active filters into query params (skipping any set to 'all') and issues an Axios GET.

e.g. GET /api/orders?warehouse=East&status=Processing&month=Q3-2025

4

FastAPI filters in memory

The route handler runs apply_filters() (warehouse / category / status) and filter_by_month() (direct month or quarter expansion) over the in-memory lists — case-insensitive, non-mutating.

5

Pydantic validates the response

The declared response_model coerces and validates the output shape before it leaves the server.

6

Vue stores raw data, derives the rest

The view stores the response in refs (e.g. allOrders, inventoryItems); computed properties derive sorted/aggregated views and the template re-renders reactively.

Cross-Cutting: The Filter System

Four filters are defined once in useFilters and applied consistently across views and endpoints. Each defaults to 'all' and is omitted from requests when inactive.

Time Period → month (e.g. 2025-09 or Q3-2025) Warehouse → warehouse Category → category Order Status → status

Inventory has no time dimension, so the month filter does not apply to /api/inventory.
