Filter Data Flow

How a filter selection in the UI becomes a filtered API response — end to end.

1

User selects filters in FilterBar

FilterBar.vue exposes 4 dropdowns — Time Period, Warehouse, Category, Order Status. Each binds to a shared ref in useFilters() composable.

// useFilters.js — singleton state shared across all views
const selectedPeriod   = ref('all')   // 'all' | 'YYYY-MM' | 'Q#-YYYY'
const selectedLocation = ref('all')   // 'all' | 'Tokyo' | 'London' | 'San Francisco'
const selectedCategory = ref('all')
const selectedStatus   = ref('all')
2

View watchers fire automatically

Every view watches the filter refs and calls its load function when any value changes.

// In each view (e.g. Orders.vue)
watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus],
  () => loadOrders()
)
3

api.js builds the HTTP request

getCurrentFilters() returns a params object. api.js appends non-"all" values as query params and calls Axios.

// api.js
getOrders(filters) {
  const params = new URLSearchParams()
  if (filters.warehouse !== 'all') params.append('warehouse', filters.warehouse)
  if (filters.month    !== 'all') params.append('month',    filters.month)
  // → GET /api/orders?warehouse=Tokyo&month=2025-01
  return axios.get(`/api/orders?${params}`)
}

Vue reactivity updates the UI

The filtered array is assigned to a ref(). Computed properties recalculate automatically (charts, stats, counts). Vue re-renders only the changed DOM nodes.

Key Patterns

Composable Singleton (Filters)

useFilters() returns the same refs every call. FilterBar mutates them; all views watch them. No Vuex/Pinia needed — the module-level refs are the store.

Raw refs + Computed Derivation

API responses land in ref([]). Every chart, stat, and table is a computed() derived from it. No manual recalculation — Vue handles cache invalidation.

Backend Filter Helpers

apply_filters() and filter_by_month() are pure functions used by every endpoint. Quarter strings (Q1-2025) expand to month lists before filtering.

Data Loaded Once at Startup

mock_data.py reads all JSON files when the server starts. All requests filter from in-memory lists — zero disk I/O per request, instant response times.

i18n + Currency Mapping

Locale stored in localStorage. Switching EN↔JA swaps translations, product/city names, and automatically converts all monetary values between USD and JPY (1:150 rate).

Custom SVG Charts

No charting library. All charts are hand-built SVG with computed coordinates. viewBox makes them responsive. Chart data is always a computed() off the raw API ref.
