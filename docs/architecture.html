System architecture

End-to-end data flow

  1. User toggles a control in FilterBar.vue; mutates a module-scoped ref in useFilters.js.
  2. The active view calls getCurrentFilters() and passes the shape to api.js.
  3. api.js builds URLSearchParams, omitting any value equal to 'all', then axios.gets the endpoint.
  4. FastAPI runs apply_filters() and/or filter_by_month() over the in-memory list.
  5. Result is validated by a Pydantic response model and serialised to JSON.
  6. The view stores the response in a ref; computed properties derive totals, sorted lists, and chart series for the template.

Filter contract

Four filters span the entire app — Period, Warehouse, Category, Order Status — passed as optional query parameters. Both client and server treat the string 'all' as “no filter.” Category and status comparisons are case-insensitive; warehouse is case-sensitive. Quarter values must match QUARTER_MAP keys exactly: Q1-2025, Q2-2025, Q3-2025, Q4-2025.

Naming mismatch worth knowing. In the UI/composable the warehouse filter is bound to selectedLocation, but getCurrentFilters() emits it as warehouse for the API. The API key must stay warehouse to match the backend contract.
