Filter Data Flow
1
User changes a filter
FilterBar.vue updates a shared ref in useFilters.js — e.g. selectedLocation.value = "Tokyo"
2
View detects the change
Each view uses watch() on the filter refs. On change it calls getCurrentFilters() and triggers a data reload.
3
API client builds the request
api.js converts the filter object to URLSearchParams and issues e.g. GET /api/orders?warehouse=Tokyo&status=processing&month=2025-09
4
FastAPI filters in memory
apply_filters() chains warehouse → category → status checks; filter_by_month() matches the YYYY-MM prefix in order_date. Pydantic validates the result.
5
Vue re-renders with new data
The axios response updates a ref(); computed properties (totals, charts, tables) recalculate automatically via Vue's reactivity system.
