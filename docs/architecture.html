Data Flow — Filter Change to UI Update
1

User changes a filter

FilterBar.vue — user selects "London" in the Location dropdown. The v-model binding writes to selectedLocation ref inside useFilters().

2

Singleton composable propagates the change

useFilters() holds module-level refs shared across every page. All views that call this composable see the updated value instantly. Each view watches getCurrentFilters() and calls its loadData() method.

3

api.js builds the HTTP request

getCurrentFilters() returns { warehouse: "London", category: "all", status: "all", month: undefined }. api.js converts non-"all" values into URLSearchParams and calls axios.get("http://localhost:8001/api/dashboard/summary?warehouse=London").

4

FastAPI route receives the request

main.py extracts warehouse="London" from query params. apply_filters(inventory_items, warehouse="London") performs an in-memory list comprehension, keeping only items where item["warehouse"] == "London". Same for orders.

5

Aggregation over filtered data

Dashboard summary calculates: total_inventory_value = sum(qty * unit_cost), low_stock_items (qty ≤ reorder_point), pending_orders (status in Processing/Backordered). All computed in Python from the filtered in-memory list.

6

Pydantic serializes the response

FastAPI uses Pydantic v2 models to validate and serialize the result dict to JSON. Response headers include Content-Type: application/json. CORS headers allow the Vue dev server on port 3000 to receive it.

7

Vue reactive state updates — DOM re-renders

summary.value = response.data triggers Vue reactivity. Computed properties recalculate, v-for loops diff and patch the DOM. The user sees updated KPI numbers with no page reload.
