Data flow — a filtered request

  1. A view (e.g. Orders.vue) reads the global filter state from useFilters() — warehouse, category, status, period.
  2. It calls an api.js function (e.g. getOrders(filters)), which serializes filters into a query string.
  3. Axios sends an HTTP request to http://localhost:8001/api/orders?warehouse=…&status=…&month=….
  4. FastAPI matches the route in main.py; query params arrive as typed, optional arguments.
  5. The handler runs apply_filters() then filter_by_month() over the in-memory list (plain list comprehensions; quarters expand via QUARTER_MAP).
  6. The result is validated/serialized through a Pydantic response_model and returned as JSON.
  7. The view stores raw data in refs; derived values (totals, badges) come from computed properties and render in the table/charts.

Key characteristics

In-memory only

JSON loaded once at startup. Writes (tasks, purchase orders) live in process memory and reset on restart — no database.

No authentication

All endpoints are public; useAuth returns a mock user. Not production-hardened.

Direct API calls

No Vite proxy — the frontend hits a hardcoded http://localhost:8001/api; backend CORS is fully open.

Computed at request time

Dashboard, reports and backlog flags are derived per request from the filtered lists, not stored.
