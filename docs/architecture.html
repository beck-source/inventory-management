Filter Data Flow

1. FilterBar.vue
  • selectedPeriod (v-model)
  • selectedLocation (v-model)
  • selectedCategory (v-model)
  • selectedStatus (v-model)
2. useFilters.js (Singleton)
  • Reactive refs shared globally
  • watch() triggers loadData()
  • getCurrentFilters() builds params
  • resetFilters() clears all to 'all'
3. api.js (Axios)
  • warehouse=San Francisco
  • category=circuit boards
  • status=delivered
  • month=2025-01 or Q1-2025
4. FastAPI Backend
  • apply_filters() — warehouse, category, status
  • filter_by_month() — month or quarter
  • Case-insensitive matching
  • No mutation of source data
1
User selects filters in FilterBar
Sticky filter bar at top of every page. Four dropdowns bound via v-model to composable refs. Selecting "All" disables that filter dimension.
2
useFilters composable triggers watchers
All views watch the shared filter refs. Any change fires loadData(), calling getCurrentFilters() which maps UI values to API query params.
// useFilters.js — singleton pattern const getCurrentFilters = () => ({ warehouse: selectedLocation.value, // "San Francisco" | "all" category: selectedCategory.value, // "circuit boards" | "all" status: selectedStatus.value, // "delivered" | "all" month: selectedPeriod.value // "2025-01" | "Q1-2025" | "all" })
3
api.js dispatches Axios request with query params
Centralized API client builds URLSearchParams from filter object and sends GET request. Dashboard makes 4 parallel requests via Promise.all.
GET /api/orders?warehouse=San+Francisco&category=circuit+boards&status=delivered&month=2025-01
4
FastAPI filters in-memory data and returns JSON
apply_filters() runs warehouse → category → status sequentially on in-memory list copies. filter_by_month() then handles date dimension. Pydantic validates the response.
def get_orders(warehouse=None, category=None, status=None, month=None): result = apply_filters(orders, warehouse, category, status) result = filter_by_month(result, month) return result # validated via response_model=List[Order]
5
Vue refs update and computed properties recalculate
Response stored in reactive ref (e.g., allOrders). Computed properties derived from the ref auto-recalculate. Vue re-renders the template diff automatically.

Key Architectural Patterns

Frontend Patterns
Singleton Filter State
useFilters.js exports a single shared reactive state. All 7 views import the same instance, so filter changes instantly propagate everywhere with no prop drilling.
Raw Refs + Computed Derived Data
Source data stored in plain refs (allOrders, inventoryItems). Charts and summaries are computed properties — automatically reactive and never stale.
Parallel API Fetching
Dashboard uses Promise.all() to fire 4 requests simultaneously. Filter changes trigger a full reload to keep server and client in sync.
Custom SVG Charts
No charting library. Donut charts (order health) and bar charts (inventory by category) are hand-written SVG + CSS, keeping the bundle lean.
Backend Patterns
Immutable In-Memory Filtering
Source data loaded once at startup. Filters operate on list copies via list comprehensions — never mutating the global data. Fast for demo-scale datasets.
Pydantic v2 Validation
Every endpoint declares a response_model. Pydantic serializes and validates all output, ensuring the frontend always receives correctly typed JSON.
Case-Insensitive Matching
All string filter comparisons use .lower() normalization. "Circuit Boards" and "circuit boards" both match. Period and status filters skip when value is 'all'.
Computed Dashboard KPIs
Dashboard summary endpoint re-calculates KPIs on every request from the already-filtered subset: inventory value, low stock count, pending orders, backlog total.
