System Documentation

Factory Inventory Management — Architecture

Full-stack demo built with Vue 3 (Composition API) on the frontend and Python FastAPI on the backend. Mock data lives in JSON files loaded into memory on startup — no database. Four global filters (period, warehouse, category, status) propagate from the UI through a composable, into the API client, and into in-memory filtering on the server.

Filter Data Flow

How a single dropdown change reaches the server and returns filtered data.

01
FilterBar.vue
v-model on selects
02
useFilters()
reactive refs
03
api.js
URLSearchParams → axios
04
FastAPI route
apply_filters + filter_by_month
05
Vue computed
render KPIs / tables
User selects Tokyo + Sensors + Q1-2025 + Delivered
useFilters → getCurrentFilters() → { warehouse: "Tokyo", category: "sensors", status: "delivered", month: "Q1-2025" }
api.js → GET /api/orders?warehouse=Tokyo&category=sensors&status=delivered&month=Q1-2025
main.py → apply_filters(orders, …) → filter_by_month(…, "Q1-2025") → expands to [2025-01, 2025-02, 2025-03]
Response → List[Order] (Pydantic validated) → Vue computed props → rendered
