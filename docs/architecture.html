Request Data Flow

1
User selects a filter
FilterBar.vue → useFilters() composable (shared singleton state)
2
View triggers API call
api.js → getCurrentFilters() → builds URLSearchParams (skips "all" values)
3
Axios sends HTTP GET
GET /api/orders?warehouse=Tokyo&status=Delivered
4
FastAPI endpoint receives request
main.py → apply_filters() → filter_by_month() → in-memory array filtering
5
Pydantic serializes response
response_model=List[Order] validates and serializes filtered data → JSON
6
Vue updates reactive state
orders.value = response.data → computed properties re-derive → template re-renders

Key Design Patterns

Singleton Filter Composable

useFilters() uses module-level refs — all views share the same filter state without prop drilling.

In-Memory Filtering

JSON files loaded once at startup. All filters create new arrays — no mutations. "all" acts as a skip sentinel.

Custom SVG Charts

No chart library. Donut charts use stroke-dasharray; bar charts use CSS Grid. Lightweight but manual.

Modal-Driven Detail Views

No dedicated routes for detail pages. Clicking a table row opens a modal — parent manages modal state.
