Factory Inventory Management System

System architecture, tech stack, and data flow overview.

High-level architecture

Vue 3 client 3080

Composition API, Vite dev server. Views render filters, tables, and charts.

↔

FastAPI server 8001

REST endpoints. CORS enabled. Filters and validates data on each request.

↔

In-memory mock data

JSON files loaded at startup into Python objects. No database.

Tech stack

Frontend

  • Vue 3 (Composition API)
  • Vue Router 4
  • Vite 5 (dev server and build)
  • Axios for HTTP calls
  • Custom SVG charts, CSS Grid layouts

Backend

  • Python 3.11+
  • FastAPI
  • Pydantic models for validation
  • Uvicorn ASGI server
  • uv for dependency management

Data

  • Static JSON files in server/data/
  • Loaded into memory via mock_data.py
  • No persistent database
  • Generated by generate_data.py

Testing and tooling

  • pytest + pytest-cov (backend)
  • FastAPI TestClient
  • ESLint/build checks (frontend)
  • Playwright MCP for browser testing

Request data flow

1. Filter UI

FilterBar component sets Time period, Warehouse, Category, Order status.

→

2. api.js

Axios client builds query params and calls the FastAPI endpoint.

→

3. FastAPI route

Endpoint filters in-memory lists by warehouse, category, status, month.

→

4. Pydantic validation

Filtered results are validated/serialized against response models.

→

5. Computed properties

Vue refs hold raw data; computed properties derive view-specific values.

Key patterns

Filter system

  • Four filters: Time period, Warehouse, Category, Order status
  • Applied globally via query params on every relevant endpoint
  • Inventory has no time dimension, so month filtering does not apply there

Reactivity model

  • Raw API data held in refs (allOrders, inventoryItems)
  • View-specific values derived via computed properties
  • Keeps filtering logic out of templates
