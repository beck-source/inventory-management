Data Flow — Filter Change

1 · User Interaction
FilterBar dropdown changed
User selects a new value — e.g. Period changes from "all" to "2025-01".
<select v-model="selectedPeriod"> ← FilterBar.vue
2 · Reactive State
useFilters() singleton updates
The shared composable holds filter refs. Any view watching them is notified automatically.
// useFilters.js (singleton) const selectedPeriod = ref('all') const selectedLocation = ref('all') const selectedCategory = ref('all') const selectedStatus = ref('all')
3 · Watch Trigger
Page-level watcher fires loadData()
Every view watches all four filter refs. On any change, loadData() re-fetches from the API.
watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus], () => loadData() )
4 · HTTP Request
api.js builds and sends the request
getCurrentFilters() serialises the filter state into URL query params sent via Axios.
GET http://localhost:8001/api/orders?month=2025-01&warehouse=all&category=all
5 · Backend Filtering
FastAPI filters in-memory data
apply_filters() narrows by warehouse/category/status. filter_by_month() handles both YYYY-MM and quarter strings (Q1-2025). Returns validated Pydantic models.
def filter_by_month(items, month): # matches "2025-01" or "Q1-2025", "Q2-2025" … return [i for i in items if matches(i.order_date, month)]
6 · UI Re-render
Computed properties recalculate, DOM updates
Raw data refs update, which invalidates computed properties. Vue re-renders only the affected chart segments, KPI cards, and table rows.
const statusData = computed(() => countByStatus(allOrders.value)) const orderHealthMetrics = computed(() => calcOnTimeRate(allOrders.value)) const topProducts = computed(() => top12ByRevenue(allOrders.value)) const categoryData = computed(() => groupByCategory(inventoryItems.value))

Global State

useFilters() — filter state

selectedPeriod ref · 'all' | 'YYYY-MM' | 'Q1-2025'
selectedLocation ref · 'all' | 'San Francisco' | 'London' | 'Tokyo'
selectedCategory ref · 'all' | 'circuit boards' | 'sensors' | …
selectedStatus ref · 'all' | 'delivered' | 'shipped' | …
getCurrentFilters() returns plain object for api.js
resetFilters() resets all refs to 'all'

 Orders with full 4-filter support

 KPI summary — inventory value, low stock, pending orders, backlog count, orders value

GET /api/backlog — Backlog items — includes has_purchase_order flag
POST /api/purchase-orders — Create a new purchase order for a backlog item
GET /api/purchase-orders/{backlog_id} — Get PO linked to a backlog item
GET /api/spending/summary — Total spend by category (procurement, operational, labor, overhead)
GET /api/spending/monthly — Monthly spending breakdown across 12 months
GET /api/spending/categories — Spending aggregated by category
GET /api/spending/transactions — Recent transaction list
GET /api/reports/quarterly — Quarterly performance — Q1–Q4 2025
GET /api/reports/monthly-trends — Month-over-month trend data
GET /api/tasks — All tasks (todo list)
POST /api/tasks — Create a new task
PATCH /api/tasks/{id} — Toggle task completion
DELETE /api/tasks/{id} — Delete a task
