Data Flow — Inventory Page Load

View mounts and reads filter state
client/src/views/Inventory.vue → onMounted() calls loadInventory(), which pulls current filter values from useFilters.getCurrentFilters().
API client builds the request
client/src/api.js → getInventory(filters) serializes { warehouse, category } to query params (skipping any value of 'all'), then issues axios.get('/api/inventory?...').
FastAPI handles and filters
server/main.py → GET /api/inventory receives query params, calls apply_filters(inventory_items, warehouse, category) to filter the in-memory list (case-insensitive on category).
Pydantic validates the response
Endpoint declares response_model=List[InventoryItem], so each row is validated and serialized through the Pydantic model before leaving the server.
Vue renders reactively
Response lands in the inventoryItems ref. A computed property filteredItems further narrows by searchQuery client-side; v-for renders the table keyed by sku.

Time period accepts month strings (2025-01) or quarter shorthand (Q1-2025); the backend expands quarters to a list of months. Inventory has no time dimension by design, and reports return pre-aggregated values.

Key Patterns

Single source of filter state

client/src/composables/useFilters.js
A module-level singleton exposes refs selectedPeriod, selectedLocation, selectedCategory, selectedStatus. Every view imports the same instance, so the FilterBar in App.vue and downstream views stay in sync without Pinia.

Stateless backend, stateful frontend

server/mock_data.py loads JSON once
JSON files load into module-level lists at import; endpoints filter copies via apply_filters() rather than mutating originals. Mutations (e.g. purchase order creation) live only in process memory and reset on restart.

Raw refs + computed derivations

Vue reactivity convention
Views hold raw API data in refs (inventoryItems, allOrders) and derive everything visible through computed properties — sort order, search filtering, grouping. Keeps re-renders cheap and logic colocated.

Validation as a contract

Pydantic response_model
Every endpoint declares a response_model, which doubles as validation and OpenAPI schema. Updating a JSON file without updating the matching Pydantic model is the most common source of 500s in this repo.
