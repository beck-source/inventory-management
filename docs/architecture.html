Filter Data Flow
Trace of a single user filter change — e.g., selecting Warehouse: Tokyo
1
User selects filter in FilterBar.vue
v-model updates selectedLocation.value = "Tokyo" (useFilters composable singleton)
2
Reactive watch fires in the active view (e.g., Orders.vue)
watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus], () => loadOrders())
3
View calls getCurrentFilters() and invokes api.js
const filters = getCurrentFilters() // { warehouse: "Tokyo", category: "all", status: "all", month: "all" } api.getOrders(filters)
4
api.js builds query string and fires HTTP request
GET http://localhost:8001/api/orders?warehouse=Tokyo // "all" values are omitted from params
5
FastAPI route receives request, runs apply_filters()
filtered = [item for item in orders if item.get("warehouse") == "Tokyo"] # then filter_by_month() if month param present
6
Pydantic validates output, response sent as JSON array
response_model=List[Order] → [{id, order_number, customer, status, total_value, ...}, ...]
7
Vue view assigns response to local ref, template re-renders
orders.value = response.data // v-for="order in orders" re-renders with Tokyo orders only

Known Constraints
Filter Limitations
  • Inventory has no time dimension — month filter does not apply
  • Demand and backlog endpoints take no filters at all
  • Quarter filter (Q1–Q4) expands to 3 months via QUARTER_MAP in main.py
Revenue Goals (hardcoded)
  • Single month target: $800K
  • YTD target (all months): $9.6M
  • Stock status: Low if qty ≤ reorder_point, Adequate if ≤ reorder_point × 1.5
Data Mutation
  • Task CRUD and Purchase Order creation mutate in-memory lists only
  • Changes are lost on server restart — no persistence layer
Frontend Patterns
  • Use unique keys in v-for — sku, order_number, month (never index)
  • Validate dates before calling .getMonth()
  • Pydantic models must be updated when JSON structure changes
