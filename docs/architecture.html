Architecture Reference

Factory Inventory Management System

A demo full-stack application for inventory tracking, order management, demand forecasting, backlog monitoring, and spending analytics. Vue 3 single-page client, FastAPI service, and JSON-file mock data — no database.

Tech stack

Three layers, each with a small, deliberate dependency surface.

Client

Vue 3 + Vite

  • vue ^3.4 — Composition API only
  • vue-router ^4.3 — six page routes
  • axios ^1.6 — HTTP client
  • vite ^5.2 — dev server on port 3000
Server

FastAPI + Pydantic

  • fastapi — routing, OpenAPI at /docs
  • pydantic ^2.5 — response models / validation
  • uvicorn — ASGI server on port 8001
  • uv — Python toolchain & venv
Data

JSON files (in-memory)

  • Loaded once at import in mock_data.py
  • Files in server/data/*.json
  • Mutations are non-persistent
  • Restart server to reload from disk

System architecture

Browser → Vite dev server → FastAPI → in-memory JSON. CORS is fully open in dev.

API surface

All endpoints are GET. Filters that respect the global filter bar are marked filtered.

Method Path Filters Notes
GET /api/inventory warehouse, category No time dimension — month does not apply.
GET /api/inventory/{id} none 404 on missing.
GET /api/orders warehouse, category, status, month Month accepts YYYY-MM or Q1-2025…Q4-2025.
GET /api/orders/{id} none 404 on missing.
GET /api/dashboard/summary all four Returns inventory value, low-stock count, pending-order count, total backlog, total order value.
GET /api/demand none Forecasted vs. current demand per SKU.
GET /api/backlog none Injects has_purchase_order by joining purchase_orders.json.
GET /api/spending/{summary, monthly, categories, transactions} none Four sibling endpoints under spending.
GET /api/reports/{quarterly, monthly-trends} none Aggregates orders by quarter or month.

Client state & layout

Three composables hold global state. Everything else is local view state.

Composables

  • useFilters — singleton refs for Period, Location, Category, Status. Same instance across all callers.
  • useI18n — en / ja locales, persisted in localStorage. Currency derives from locale (USD or JPY).
  • useAuth — mock current user, locale-aware fields, sample tasks. No real auth.

Routes (main.js)

  • / → Dashboard
  • /inventory → Inventory
  • /orders → Orders
  • /demand → Demand
  • /spending → Spending
  • /reports → Reports

Server modules

  • main.py — routes, Pydantic models, filter helpers, QUARTER_MAP.
  • mock_data.py — loads each JSON file into a module-level list at import time.
  • generate_data.py — one-off script to regenerate sample data.

Data files (server/data/)

  • inventory.json
  • orders.json
  • demand_forecasts.json
  • backlog_items.json
  • purchase_orders.json
  • spending.json (summary / monthly / category)
  • transactions.json

Run & verify

Two processes, two ports. Tests live in their own folder with a dedicated pytest.ini.

Backend — port 8001

  • cd server && uv venv && uv sync
  • uv run python main.py
  • Open http://localhost:8001/docs for Swagger UI.

Frontend — port 3000

  • cd client && npm install
  • npm run dev
  • Open http://localhost:3000.

Tests

  • cd tests && uv run pytest -v
  • One file: uv run pytest backend/test_inventory.py -v
  • One test: …::TestInventoryEndpoints::test_get_all_inventory

Production build

  • cd client && npm run build → client/dist/
  • No build step on the server — FastAPI serves Python directly.
