Request / Data Flow

1

User selects a filter

FilterBar.vue updates the shared composable — useFilters() holds four refs: selectedPeriod, selectedLocation, selectedCategory, selectedStatus.

2

View component reacts

Each view watches the filter refs. On change it calls getCurrentFilters() to get { warehouse, category, status, month } and dispatches an API call.

3

api.js builds the HTTP request

Axios appends non-all filters as query params:
GET /api/orders?warehouse=Tokyo&category=sensors&month=2025-03

4

FastAPI endpoint filters in-memory data

apply_filters(items, warehouse, category, status) does case-insensitive matching. filter_by_month() matches ISO date strings or resolves quarter aliases (e.g. Q1-2025 → Jan/Feb/Mar). Pydantic validates and serializes the response.

5

Response arrives in the view

Raw data stored in a ref (e.g. allOrders). computed properties derive KPIs, totals, and chart data. Vue re-renders only affected DOM nodes.

API Endpoints

Inventory

GET /api/inventory Filters: warehouse, category
GET /api/inventory/{item_id} Single item by ID

Orders

GET /api/orders Filters: warehouse, category, status, month
GET /api/orders/{order_id} Single order by ID

Dashboard

GET /api/dashboard/summary Filters: all four. Returns KPIs.

Demand & Backlog

GET /api/demand No filters — full forecast list
GET /api/backlog No filters — adds has_purchase_order flag

Spending

GET /api/spending/summary Totals & % change by category
GET /api/spending/monthly Month-by-month breakdown
GET /api/spending/categories Spending by category
GET /api/spending/transactions Recent transaction list

Reports

GET /api/reports/quarterly Quarterly rollup from orders
GET /api/reports/monthly-trends Month-over-month trends from orders

Purchase Orders

POST /api/purchase-orders Create PO linked to backlog item
GET /api/purchase-orders/{backlog_item_id} Get PO by backlog item ID
