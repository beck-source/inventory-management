Request Data Flow

1
User changes a filter
The FilterBar.vue component updates the useFilters() singleton composable. All views share this reactive state.
2
View triggers an API call
Each view calls the relevant method in client/src/api.js, passing active filter values as URLSearchParams. Filters with value 'all' are omitted from the query string.
3
FastAPI receives the request
server/main.py routes the request. Filter params are read from the query string and passed to apply_filters() and filter_by_month() utilities.
4
In-memory data is filtered
Data loaded by mock_data.py at startup is filtered on copies — originals are never mutated. A QUARTER_MAP maps Q1–Q4 labels to month lists.
5
Pydantic validates the response
FastAPI serialises the result through Pydantic models (InventoryItem, Order, DemandForecast, etc.) before returning JSON.
6
Vue renders from computed properties
Raw response is stored in a ref (e.g. allOrders). Computed properties derive filtered, sorted, and formatted views of that data for the template.

Filter System

Time Period
Monthly (Jan–Dec 2025) or quarterly (Q1–Q4). Maps to month query param.
Orders / Dashboard
Location (Warehouse)
North, South, East warehouses — or All.
Inventory / Orders / Dashboard
Category
Circuit Boards, Sensors, Actuators, Controllers, Power Supplies.
Inventory / Orders / Dashboard
Order Status
Delivered, Shipped, Processing, Backordered.
Orders / Dashboard

Note: Inventory does not support the Time Period filter (no time dimension on stock levels).
