Data Flow — Filter Request Example

1
User selects filters in FilterBar.vue
Warehouse → "San Francisco"  ·  Period → "2025-03"  ·  Status → "Delivered"
2
useFilters() composable updates shared state
Refs update: selectedLocation, selectedPeriod, selectedStatus — all views watching these refs react automatically.
3
View calls api.js with current filters
getCurrentFilters() returns { warehouse, month, status, category } — passed into the relevant API method.
4
api.js constructs URL and fires axios GET
URLSearchParams skips "all" values. Result: GET /api/orders?warehouse=San+Francisco&month=2025-03&status=Delivered
5
FastAPI endpoint receives query params
apply_filters() chains warehouse → category → status checks. filter_by_month() handles YYYY-MM strings and Q1–Q4 shorthand.
6
Filtered data validated by Pydantic, returned as JSON
Response is List[Order] — Pydantic serializes each item. axios resolves response.data.
7
Vue view updates reactively
Raw data stored in ref([]). Computed properties (tables, charts, KPIs) recalculate automatically.

API Endpoints

Method Path Query Params Description
GET /api/inventory warehouse category All inventory items
GET /api/inventory/{id} — Single inventory item
GET /api/orders warehouse category status month All orders with full filter support
GET /api/orders/{id} — Single order detail
GET /api/demand — Demand forecasts (no filters)
GET /api/backlog — Backlog items with purchase order flag
GET /api/dashboard/summary warehouse category status month Aggregated KPI summary
GET /api/spending/summary — Total spending by cost type
GET /api/spending/monthly — Month-by-month spending breakdown
GET /api/spending/categories — Spending per product category
GET /api/spending/transactions — Recent transaction records
GET /api/reports/quarterly — Quarterly order count, revenue, fulfillment
GET /api/reports/monthly-trends — Month-over-month trend data
POST /api/purchase-orders — Create purchase order (stub)
