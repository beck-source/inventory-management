Data Flow — Request Lifecycle

What happens from a filter change to a re-rendered chart.

1

User adjusts a filter

The shared FilterBar exposes four filters: Time Period, Location (warehouse), Category, and Order Status.

2

Shared state updates

useFilters holds the selections as module-level refs — a singleton, so every view sees the same filter state instantly.

3

Active view requests data

The view calls getCurrentFilters() and passes it to an api.js method, which builds the query string.

4

axios sends the HTTP request

A GET goes to http://localhost:8001/api/… with the filters as query params.

5

FastAPI filters in memory

The endpoint runs apply_filters() (warehouse / category / status) and filter_by_month() over the in-memory lists — dashboard & report endpoints also aggregate totals.

6

Pydantic validates the response

Typed response_model classes shape and validate the JSON before it leaves the server.

7

View stores raw data, derives the rest

Results land in a ref; computed properties turn them into KPIs, table rows, and chart series.

8

Vue re-renders

Reactivity updates the DOM — stat cards, data tables, and custom SVG charts repaint with the filtered numbers.

Implementation Notes

  • Demo posture: no authentication, CORS allows all origins, and data is read-only in memory — restarting the server resets any state. Intended for local development only.
  • Routing: the app uses a small hand-written router (router.js) instead of vue-router, registering global router-link/router-view components and a reactive $route.
  • Work-in-progress endpoints: the client's api.js also calls /api/tasks and /api/purchase-orders, which the backend does not yet implement (those calls currently 404).
