Filter Data Flow
1
User selects filters
FilterBar.vue binds to the useFilters composable — a singleton that holds shared reactive state across all views.
// composables/useFilters.js — singleton reactive state selectedPeriod = 'Q1-2025' selectedLocation = 'San Francisco' selectedCategory = 'all' selectedStatus = 'Processing'
2
Composable maps to API params
getCurrentFilters() transforms UI state into query param keys expected by the backend.
// selectedLocation → warehouse, selectedPeriod → month getCurrentFilters() → { warehouse: 'San Francisco', category: 'all', status: 'Processing', month: 'Q1-2025' }
3
api.js fires HTTP request
Axios appends non-'all' params as query strings. The 'all' value is never sent to the backend.
// client/src/api.js GET /api/orders?warehouse=San Francisco&status=Processing&month=Q1-2025
4
FastAPI applies filters
apply_filters() handles warehouse/category/status with case-insensitive matching. filter_by_month() expands quarter codes into date prefixes.
# server/main.py — two-pass filtering apply_filters(orders, warehouse='San Francisco', status='Processing') filter_by_month(filtered, month='Q1-2025') # Q1-2025 → ['2025-01', '2025-02', '2025-03']
5
Pydantic validates & serialises
Filtered results are returned as List[Order]. Pydantic validates field types and serialises to JSON automatically.
# Response — validated List[Order] as JSON [ { "id": 42, "order_number": "ORD-00042", "warehouse": "San Francisco", "status": "Processing", … } ]
6
Vue reactivity updates the UI
Each view stores raw data in a ref() and exposes derived data via computed(). Only the computed re-evaluates when the ref changes.
// view setup() allOrders = ref([]) // raw, from API filteredOrders = computed(() => …) // derived, cached
