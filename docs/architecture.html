3Data Flow — Filtering Orders by Quarter

1
Frontend
User selects Q2-2025 in the filter bar. The dropdown updates a shared ref in useFilters.
client/src/components/FilterBar.vue → client/src/composables/useFilters.js
2
Frontend
View watches the ref and calls api.getOrders({ month: 'Q2-2025' }).
client/src/views/Orders.vue → client/src/api.js
3
Network
Axios issues GET http://localhost:8001/api/orders?month=Q2-2025.
4
Backend
Route handler expands Q2-2025 to ['2025-04', '2025-05', '2025-06'] via QUARTER_MAP, then filters orders by date prefix.
server/main.py — apply_filters() & filter_by_month()
5
Data
Filter operates on the in-memory list loaded from orders.json; matching dicts are validated against the Order Pydantic model.
server/mock_data.py → server/data/orders.json
6
Backend
FastAPI serializes the list to JSON and returns 200 OK.
7
Frontend
Response data is assigned to a ref; computed properties derive table rows; the view re-renders. Clicking a row opens OrderDetailModal.
client/src/views/Orders.vue
